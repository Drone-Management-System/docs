# Data Model

The CAA Registry API maps directly to PostgreSQL tables defined in
`src/db/schema.ts` and created by `drizzle/0002_caa_registry.sql`.

## Schemas

| Schema | Purpose |
| --- | --- |
| `caa_registry_api` | Registry tables owned by this service. |
| `public` | Shared `audit_log` table used for write history. |
| `remote_id_uss_api` | External certificate table read by `GET /uas/:uasId/certificates`. |

## Organization Kinds

`organization.organization_kind` must be one of:

| Value | Typical use |
| --- | --- |
| `CAA` | Civil aviation authority. |
| `USS` | UAS service supplier. |
| `OPERATOR` | UAS operator organization. |
| `SERVICE_PROVIDER` | Service provider organization. |
| `OTHER` | Fallback when the organization does not fit another category. |

The API defaults omitted organization kind values to `OTHER`.

## `caa_registry_api.operator_type`

Reference table for classifying operators.

| Column | Type | Required | Notes |
| --- | --- | --- | --- |
| `operator_type_id` | `smallint` | Yes | Primary key supplied by client. |
| `code` | `text` | Yes | Unique code. |
| `description` | `text` | No | Human-readable description. |

## `caa_registry_api.user_type`

Reference table for classifying registry users.

| Column | Type | Required | Notes |
| --- | --- | --- | --- |
| `user_type_id` | `smallint` | Yes | Primary key supplied by client. |
| `code` | `text` | Yes | Unique code. |
| `description` | `text` | No | Human-readable description. |

## `caa_registry_api.organization`

Organizations represent CAAs, USSs, operators, service providers, and other
registry participants.

| Column | Type | Required | Notes |
| --- | --- | --- | --- |
| `org_id` | `uuid` | Yes | Primary key generated by the database. |
| `org_name` | `text` | Yes | Unique organization name. |
| `organization_kind` | `text` | Yes | Enum-like value with database check constraint. Defaults to `OTHER`. |
| `contact_email` | `text` | No | Optional contact email. |
| `contact_phone` | `text` | No | Optional contact phone. |
| `country_code` | `text` | No | Uppercased by API when supplied. |
| `created_at` | `timestamp with time zone` | Yes | Defaults to `now()`. |
| `updated_at` | `timestamp with time zone` | Yes | Defaults to `now()`, updated on `PUT`. |

Indexes and constraints:

| Name | Purpose |
| --- | --- |
| `organization_org_name_unique` | Prevents duplicate organization names. |
| `organization_kind_idx` | Speeds filtering by organization kind. |
| `organization_kind_valid` | Restricts `organization_kind` to supported values. |

## `caa_registry_api.user`

Registry users represent people in the registry. A user can belong to an
organization and a user type, but both links are optional and set to `null`
if the referenced row is deleted.

| Column | Type | Required | Notes |
| --- | --- | --- | --- |
| `user_id` | `uuid` | Yes | Primary key generated by the database. |
| `org_id` | `uuid` | No | References `organization.org_id`, `on delete set null`. |
| `user_type_id` | `smallint` | No | References `user_type.user_type_id`, `on delete set null`. |
| `full_name` | `text` | Yes | Required display name. |
| `email` | `text` | Yes | Unique and lowercased by API. |
| `phone` | `text` | No | Optional phone. |
| `created_at` | `timestamp with time zone` | Yes | Defaults to `now()`. |
| `updated_at` | `timestamp with time zone` | Yes | Defaults to `now()`, updated on `PUT`. |

Indexes and constraints:

| Name | Purpose |
| --- | --- |
| `user_email_unique` | Prevents duplicate user emails. |
| `user_org_id_idx` | Speeds organization filtering. |
| `user_user_type_id_idx` | Speeds user type filtering. |
| `user_email_idx` | Speeds email search and lookup paths. |

## `caa_registry_api.operator`

Operators represent approved operating entities linked to organizations.

| Column | Type | Required | Notes |
| --- | --- | --- | --- |
| `operator_id` | `uuid` | Yes | Primary key generated by the database. |
| `org_id` | `uuid` | Yes | References `organization.org_id`, `on delete restrict`. |
| `primary_user_id` | `uuid` | No | References `user.user_id`, `on delete set null`. Must belong to the same org when the user has an org. |
| `operator_type_id` | `smallint` | No | References `operator_type.operator_type_id`, `on delete set null`. |
| `approval_reference` | `text` | No | Unique approval reference. |
| `approval_valid_from` | `date` | No | Date-only string in API. |
| `approval_valid_to` | `date` | No | Must be on or after `approval_valid_from` when both exist. |
| `created_at` | `timestamp with time zone` | Yes | Defaults to `now()`. |
| `updated_at` | `timestamp with time zone` | Yes | Defaults to `now()`, updated on `PUT`. |

Indexes and constraints:

| Name | Purpose |
| --- | --- |
| `operator_approval_reference_unique` | Prevents duplicate approval references. |
| `operator_org_id_idx` | Speeds organization filtering. |
| `operator_primary_user_id_idx` | Speeds primary user joins. |
| `operator_operator_type_id_idx` | Speeds operator type filtering. |
| `operator_approval_dates_valid` | Enforces valid approval windows. |

## `caa_registry_api.uas`

UAS records represent registered unmanned aircraft systems. Each UAS belongs to
exactly one operator.

| Column | Type | Required | Notes |
| --- | --- | --- | --- |
| `uas_id` | `uuid` | Yes | Primary key generated by the database. |
| `operator_id` | `uuid` | Yes | References `operator.operator_id`, `on delete restrict`. |
| `registration_id` | `text` | Yes | Unique public registration identifier. |
| `serial_number` | `text` | Yes | Unique serial number. |
| `model_name` | `text` | No | Optional model name. |
| `remote_id_capable` | `boolean` | Yes | Defaults to `true`. |
| `created_at` | `timestamp with time zone` | Yes | Defaults to `now()`. |
| `updated_at` | `timestamp with time zone` | Yes | Defaults to `now()`, updated on `PUT`. |

Indexes and constraints:

| Name | Purpose |
| --- | --- |
| `uas_registration_id_unique` | Prevents duplicate registration IDs. |
| `uas_serial_number_unique` | Prevents duplicate serial numbers. |
| `uas_operator_id_idx` | Speeds operator filtering and joins. |

## `caa_registry_api.ua`

UA records represent individual aircraft under a UAS registration.

| Column | Type | Required | Notes |
| --- | --- | --- | --- |
| `ua_id` | `uuid` | Yes | Primary key generated by the database. |
| `uas_id` | `uuid` | Yes | References `uas.uas_id`, `on delete cascade` at database level. |
| `aircraft_type` | `text` | Yes | Required aircraft classification. |
| `propulsion_type` | `text` | No | Optional propulsion classification. |
| `mass_kg` | `numeric(8,2)` | No | Must be non-negative when supplied. |
| `max_airspeed_mps` | `numeric(7,2)` | No | Must be non-negative when supplied. |
| `max_altitude_m` | `numeric(8,2)` | No | Must be non-negative when supplied. |
| `endurance_minutes` | `integer` | No | Must be positive when supplied. |
| `created_at` | `timestamp with time zone` | Yes | Defaults to `now()`. |
| `updated_at` | `timestamp with time zone` | Yes | Defaults to `now()`, updated on `PUT`. |

Indexes and constraints:

| Name | Purpose |
| --- | --- |
| `ua_uas_id_idx` | Speeds UAS filtering and joins. |
| `ua_mass_non_negative` | Prevents negative mass. |
| `ua_max_airspeed_non_negative` | Prevents negative max airspeed. |
| `ua_max_altitude_non_negative` | Prevents negative max altitude. |
| `ua_endurance_positive` | Prevents zero or negative endurance. |

## `public.audit_log`

The audit table records mutable registry operations.

| Column | Type | Required | Notes |
| --- | --- | --- | --- |
| `audit_id` | `uuid` | Yes | Primary key generated by the database. |
| `entity_type` | `text` | Yes | Examples: `organization`, `user`, `operator`, `uas`, `ua`, `operator_type`, `user_type`. |
| `entity_id` | `text` | Yes | Entity ID as text. |
| `action` | `text` | Yes | `create`, `update`, or `delete`. |
| `actor` | `text` | Yes | Email, username, subject, or `system`. |
| `correlation_id` | `text` | No | Column exists but route handlers do not currently populate it. |
| `created_at` | `timestamp with time zone` | Yes | Defaults to `now()`. |
| `before_json` | `jsonb` | No | Snapshot before update or delete. |
| `after_json` | `jsonb` | No | Snapshot after create, update, or delete. |

Indexes:

| Name | Purpose |
| --- | --- |
| `audit_log_entity_idx` | Speeds entity history queries. |
| `audit_log_created_at_idx` | Speeds recent audit listing. |

## Relationship Summary

| Parent | Child | Delete behavior |
| --- | --- | --- |
| `organization` | `user` | User `org_id` is set to `null`. |
| `organization` | `operator` | Delete is restricted. |
| `user_type` | `user` | User `user_type_id` is set to `null`. |
| `operator_type` | `operator` | Operator `operator_type_id` is set to `null`. |
| `user` | `operator.primary_user_id` | Operator primary user is set to `null`. |
| `operator` | `uas` | Delete is restricted. API also checks and returns `operator_has_uas`. |
| `uas` | `ua` | Database cascades, but API delete checks and returns `uas_has_aircraft`. |

The API is more conservative than the database for some deletes. For example,
the database could cascade UA rows when a UAS is deleted, but the API blocks
that operation when linked UA rows exist.
