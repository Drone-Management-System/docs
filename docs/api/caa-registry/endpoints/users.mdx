# Registry Users

Registry users represent people associated with organizations and registry
workflows. They map to `caa_registry_api.user`.

## Permissions

| Operation | Permission |
| --- | --- |
| List or fetch users | `users.read` |
| Create or replace users | `users.write` |
| Delete users | `users.delete` |
| Read user history | `audit.read` |

## User Resource

```json
{
  "user_id": "afdf33ff-3db4-4972-b89a-2ea1f9e0a708",
  "org_id": "8c3198d0-6f9d-4a41-b276-6b60da0bdf54",
  "user_type_id": 1,
  "zitadel_user_id": "298183789123456789",
  "zitadel_status": "active",
  "zitadel_deleted_at": null,
  "zitadel_last_synced_at": "2026-06-03T12:00:00.000Z",
  "national_id_number": "784-1990-1234567-1",
  "national_id_country_code": "AE",
  "national_id_type": "emirates_id",
  "full_name": "Amina Hassan",
  "email": "amina@example.test",
  "phone": "+971500000002",
  "created_at": "2026-05-14T08:00:00.000Z",
  "updated_at": "2026-05-14T08:00:00.000Z"
}
```

| Field | Type | Notes |
| --- | --- | --- |
| `user_id` | UUID string | Generated by the database. |
| `org_id` | UUID string or `null` | Optional organization link. |
| `user_type_id` | Smallint or `null` | Optional user type link. |
| `zitadel_user_id` | String or `null` | Compatibility mirror of the ZITADEL provider user ID. Generic provider links live in `user_identity`. |
| `zitadel_status` | String | `active` or `deleted`. |
| `zitadel_deleted_at` | Timestamp or `null` | Set when ZITADEL reports `user.removed`. |
| `zitadel_last_synced_at` | Timestamp or `null` | Last time identity lifecycle state was synced from ZITADEL. |
| `national_id_number` | String or `null` | Generic national identity number. |
| `national_id_country_code` | String or `null` | Issuing country code, uppercased by API. |
| `national_id_type` | String or `null` | Identifier type, lowercased by API, such as `emirates_id`, `aadhaar`, `national_id`, or `other`. |
| `full_name` | String | Required. |
| `email` | String | Required, unique, lowercased by API. |
| `phone` | String or `null` | Optional. |
| `created_at` | Timestamp | Database generated. |
| `updated_at` | Timestamp | Updated on replacement. |

## Write Body

```json
{
  "org_id": "8c3198d0-6f9d-4a41-b276-6b60da0bdf54",
  "user_type_id": 1,
  "national_id_number": "784-1990-1234567-1",
  "national_id_country_code": "AE",
  "national_id_type": "emirates_id",
  "full_name": "Amina Hassan",
  "email": "Amina@Example.Test",
  "phone": "+971500000002"
}
```

| Field | Required | Validation |
| --- | --- | --- |
| `org_id` | No | Optional string. If supplied, the organization must exist. |
| `user_type_id` | No | Optional smallint. If supplied, the user type must exist. |
| `national_id_number` | No | Optional national identity number. If supplied, country and type are also required. |
| `national_id_country_code` | No | Optional issuing country code. Uppercased by API. |
| `national_id_type` | No | Optional identifier type. Lowercased by API. |
| `full_name` | Yes | Non-empty string. |
| `email` | Yes | Non-empty string; lowercased before write. |
| `phone` | No | Optional string; blank becomes `null`. |

Manual user writes do not accept `zitadel_*` fields. Those fields are managed by
the ZITADEL integration endpoints and system jobs.

National ID fields are generic and not UAE-specific. For example, UAE can use
`emirates_id`, India can use `aadhaar`, and other jurisdictions can use
`national_id` or another stable type value. If any national ID field is present,
all three national ID fields must be present.

## `GET /users`

<Endpoint
  method="GET"
  path="/users"
  title="List registry users"
  permission="users.read"
  success="200 OK"
  summary="Returns paginated users, optionally filtered by organization, user type, search text, email, ZITADEL state, or national ID."
/>

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "$CAA_REGISTRY_API_BASE_URL/users?org_id=8c3198d0-6f9d-4a41-b276-6b60da0bdf54&user_type_id=1&q=amina"
```

### Query Parameters

| Parameter | Default | Maximum | Notes |
| --- | --- | --- | --- |
| `limit` | `50` | `200` | Number of rows to return. |
| `offset` | `0` | `100000` | Number of rows to skip. |
| `org_id` | None | N/A | Filters by organization ID. |
| `user_type_id` | None | N/A | Filters by user type. Must be a valid smallint. |
| `q` | None | N/A | Case-insensitive match against `full_name` or `email`. |
| `email` | None | N/A | Exact email match. Lowercased before filtering. |
| `zitadel_user_id` | None | N/A | Exact ZITADEL user ID match. |
| `zitadel_status` | None | N/A | Exact identity lifecycle status: `active` or `deleted`. |
| `national_id_number` | None | N/A | Exact national ID number. Requires country code and type. |
| `national_id_country_code` | None | N/A | Exact issuing country code. Uppercased before filtering. |
| `national_id_type` | None | N/A | Exact identifier type. Lowercased before filtering. |

### Response

Rows are ordered by `created_at desc`, then `user_id desc`.

### Collision Lookup

Use exact filters to find records that would block a write. For example, if a
new active user cannot receive a national ID because a deleted user still holds
it:

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "$CAA_REGISTRY_API_BASE_URL/users?national_id_country_code=AE&national_id_type=emirates_id&national_id_number=784-1990-1234567-1&zitadel_status=deleted"
```

If a new ZITADEL user should revive an old row by email, verify the existing
row with:

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "$CAA_REGISTRY_API_BASE_URL/users?email=amina@example.test"
```

## `POST /users`

<Endpoint
  method="POST"
  path="/users"
  title="Create registry user"
  permission="users.write"
  success="201 Created"
  summary="Creates a registry user and writes a create audit event."
/>

```bash
curl -X POST "$CAA_REGISTRY_API_BASE_URL/users" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "org_id": "8c3198d0-6f9d-4a41-b276-6b60da0bdf54",
    "user_type_id": 1,
    "full_name": "Amina Hassan",
    "email": "Amina@Example.Test",
    "phone": "+971500000002"
  }'
```

### Request Body

Uses the registry user write body documented above.

### Response

Success returns the created row. The response email is lowercase:

```json
{
  "email": "amina@example.test"
}
```

### Error Responses

The service validates references before insert:

| Reference | Error when missing |
| --- | --- |
| `org_id` | `organization_not_found` |
| `user_type_id` | `user_type_not_found` |
| National ID partial input | `national_id_incomplete` |

Duplicate identity values return conflicts:

| Conflict | Detail |
| --- | --- |
| Duplicate email | `email_conflict` |
| Duplicate national ID triple | `national_id_conflict` |
| Duplicate ZITADEL ID mirror | `zitadel_user_id_conflict` |

Use `GET /users` exact filters to find the conflicting row.

## `PATCH /users/:userId/national-id`

<Endpoint
  method="PATCH"
  path="/users/:userId/national-id"
  title="Set or clear national ID"
  permission="users.write"
  success="200 OK"
  summary="Updates only the national ID fields for a registry user."
/>

This endpoint exists for maintenance workflows where a stale or deleted user row
holds a national ID that blocks the correct active record.

Set a national ID:

```bash
curl -X PATCH "$CAA_REGISTRY_API_BASE_URL/users/afdf33ff-3db4-4972-b89a-2ea1f9e0a708/national-id" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "national_id_number": "784-1990-1234567-1",
    "national_id_country_code": "AE",
    "national_id_type": "emirates_id"
  }'
```

Clear a national ID:

```bash
curl -X PATCH "$CAA_REGISTRY_API_BASE_URL/users/afdf33ff-3db4-4972-b89a-2ea1f9e0a708/national-id" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "national_id_number": null,
    "national_id_country_code": null,
    "national_id_type": null
  }'
```

At least one national ID field must be present. If setting an ID, all three
fields are required. The route writes audit action `update_national_id`.

### Collision Resolution Workflow

1. Find the row holding the conflicting national ID:

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "$CAA_REGISTRY_API_BASE_URL/users?national_id_country_code=AE&national_id_type=emirates_id&national_id_number=784-1990-1234567-1"
```

2. If the returned row is stale or incorrect, clear only its national ID:

```bash
curl -X PATCH "$CAA_REGISTRY_API_BASE_URL/users/$CONFLICTING_USER_ID/national-id" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "national_id_number": null,
    "national_id_country_code": null,
    "national_id_type": null
  }'
```

3. Retry the update on the correct user.

## `PUT /users/:userId`

<Endpoint
  method="PUT"
  path="/users/:userId"
  title="Replace registry user"
  permission="users.write"
  success="200 OK"
  summary="Replaces all writable user fields."
/>

```bash
curl -X PUT "$CAA_REGISTRY_API_BASE_URL/users/afdf33ff-3db4-4972-b89a-2ea1f9e0a708" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "org_id": "8c3198d0-6f9d-4a41-b276-6b60da0bdf54",
    "user_type_id": 1,
    "full_name": "Amina Hassan",
    "email": "amina@example.test",
    "phone": "+971500000003"
  }'
```

### Request Body

Uses the registry user write body documented above. Replacement updates all
writable fields and sets `updated_at` to database
`now()`. Omitted optional fields become `null`.

Omitting national ID fields on `PUT` clears the stored national ID fields,
matching the route's replace semantics.

<Endpoint
  method="DELETE"

### Response

Success returns the deleted row and writes an audit event.

sets that operator field to `null`.

## Validation Details

| Detail | Trigger |
| --- | --- |
| `user_id_required` | Path `userId` is missing or blank. |
| `full_name_required` | `full_name` is missing, not a string, or blank. |
| `email_required` | `email` is missing, not a string, or blank. |
| `user_type_id_invalid` | `user_type_id` is not a valid smallint. |
| `organization_not_found` | Supplied `org_id` does not exist. |
| `user_type_not_found` | Supplied `user_type_id` does not exist. |
| `user_not_found` | Requested user does not exist. |
| `user_create_failed` | Insert returned no row. |
| `user_update_failed` | Update returned no row. |
| `user_delete_failed` | Delete returned no row. |
| `national_id_payload_required` | National ID patch body did not include any national ID fields. |
| `national_id_filter_incomplete` | A national ID filter was partial. |
| `email_conflict` | Another user already has this email. |
| `national_id_conflict` | Another user already has this national ID triple. |
| `zitadel_user_id_conflict` | Another user already has this ZITADEL user ID mirror. |
