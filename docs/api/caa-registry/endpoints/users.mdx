# Registry Users

Registry users represent people associated with organizations and registry
workflows. They map to `caa_registry_api.user`.

## Permissions

| Operation | Permission |
| --- | --- |
| List or fetch users | `users.read` |
| Create or replace users | `users.write` |
| Delete users | `users.delete` |
| Read user history | `audit.read` |

## User Resource

```json
{
  "user_id": "afdf33ff-3db4-4972-b89a-2ea1f9e0a708",
  "org_id": "8c3198d0-6f9d-4a41-b276-6b60da0bdf54",
  "user_type_id": 1,
  "full_name": "Amina Hassan",
  "email": "amina@example.test",
  "phone": "+971500000002",
  "created_at": "2026-05-14T08:00:00.000Z",
  "updated_at": "2026-05-14T08:00:00.000Z"
}
```

| Field | Type | Notes |
| --- | --- | --- |
| `user_id` | UUID string | Generated by the database. |
| `org_id` | UUID string or `null` | Optional organization link. |
| `user_type_id` | Smallint or `null` | Optional user type link. |
| `full_name` | String | Required. |
| `email` | String | Required, unique, lowercased by API. |
| `phone` | String or `null` | Optional. |
| `created_at` | Timestamp | Database generated. |
| `updated_at` | Timestamp | Updated on replacement. |

## Write Body

```json
{
  "org_id": "8c3198d0-6f9d-4a41-b276-6b60da0bdf54",
  "user_type_id": 1,
  "full_name": "Amina Hassan",
  "email": "Amina@Example.Test",
  "phone": "+971500000002"
}
```

| Field | Required | Validation |
| --- | --- | --- |
| `org_id` | No | Optional string. If supplied, the organization must exist. |
| `user_type_id` | No | Optional smallint. If supplied, the user type must exist. |
| `full_name` | Yes | Non-empty string. |
| `email` | Yes | Non-empty string; lowercased before write. |
| `phone` | No | Optional string; blank becomes `null`. |

## Endpoint Catalog

| Method | Path | Purpose |
| --- | --- | --- |
| `GET` | `/users` | List and search registry users. |
| `GET` | `/users/:userId` | Fetch one registry user. |
| `POST` | `/users` | Create a registry user. |
| `PUT` | `/users/:userId` | Replace a registry user. |
| `DELETE` | `/users/:userId` | Delete a registry user. |
| `GET` | `/users/:userId/history` | Read user audit history. |

## `GET /users`

<Endpoint
  method="GET"
  path="/users"
  title="List registry users"
  permission="users.read"
  success="200 OK"
  summary="Returns paginated users, optionally filtered by organization, user type, and search text."
/>

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "$CAA_REGISTRY_API_BASE_URL/users?org_id=8c3198d0-6f9d-4a41-b276-6b60da0bdf54&user_type_id=1&q=amina"
```

### Query Parameters

| Parameter | Default | Maximum | Notes |
| --- | --- | --- | --- |
| `limit` | `50` | `200` | Number of rows to return. |
| `offset` | `0` | `100000` | Number of rows to skip. |
| `org_id` | None | N/A | Filters by organization ID. |
| `user_type_id` | None | N/A | Filters by user type. Must be a valid smallint. |
| `q` | None | N/A | Case-insensitive match against `full_name` or `email`. |

### Response

Rows are ordered by `created_at desc`, then `user_id desc`.

## `POST /users`

<Endpoint
  method="POST"
  path="/users"
  title="Create registry user"
  permission="users.write"
  success="201 Created"
  summary="Creates a registry user and writes a create audit event."
/>

```bash
curl -X POST "$CAA_REGISTRY_API_BASE_URL/users" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "org_id": "8c3198d0-6f9d-4a41-b276-6b60da0bdf54",
    "user_type_id": 1,
    "full_name": "Amina Hassan",
    "email": "Amina@Example.Test",
    "phone": "+971500000002"
  }'
```

### Request Body

Uses the registry user write body documented above.

### Response

Success returns the created row. The response email is lowercase:

```json
{
  "email": "amina@example.test"
}
```

### Error Responses

The service validates references before insert:

| Reference | Error when missing |
| --- | --- |
| `org_id` | `organization_not_found` |
| `user_type_id` | `user_type_not_found` |

## `PUT /users/:userId`

<Endpoint
  method="PUT"
  path="/users/:userId"
  title="Replace registry user"
  permission="users.write"
  success="200 OK"
  summary="Replaces all writable user fields."
/>

```bash
curl -X PUT "$CAA_REGISTRY_API_BASE_URL/users/afdf33ff-3db4-4972-b89a-2ea1f9e0a708" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "org_id": "8c3198d0-6f9d-4a41-b276-6b60da0bdf54",
    "user_type_id": 1,
    "full_name": "Amina Hassan",
    "email": "amina@example.test",
    "phone": "+971500000003"
  }'
```

### Request Body

Uses the registry user write body documented above. Replacement updates all
writable fields and sets `updated_at` to database
`now()`. Omitted optional fields become `null`.

## `DELETE /users/:userId`

<Endpoint
  method="DELETE"
  path="/users/:userId"
  title="Delete registry user"
  permission="users.delete"
  success="200 OK"
  summary="Deletes a registry user and returns the deleted row."
/>

```bash
curl -X DELETE "$CAA_REGISTRY_API_BASE_URL/users/afdf33ff-3db4-4972-b89a-2ea1f9e0a708" \
  -H "Authorization: Bearer $TOKEN"
```

### Response

Success returns the deleted row and writes an audit event.

If an operator uses this user as `primary_user_id`, database foreign-key behavior
sets that operator field to `null`.

## `GET /users/:userId/history`

<Endpoint
  method="GET"
  path="/users/:userId/history"
  title="List registry user audit history"
  permission="audit.read"
  success="200 OK"
  summary="Returns audit rows for one registry user."
/>

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "$CAA_REGISTRY_API_BASE_URL/users/afdf33ff-3db4-4972-b89a-2ea1f9e0a708/history?limit=100"
```

Requires `audit.read`. Returns audit rows for `entity_type = "user"` and the
supplied `user_id`, ordered newest first.

## Validation Details

| Detail | Trigger |
| --- | --- |
| `user_id_required` | Path `userId` is missing or blank. |
| `full_name_required` | `full_name` is missing, not a string, or blank. |
| `email_required` | `email` is missing, not a string, or blank. |
| `user_type_id_invalid` | `user_type_id` is not a valid smallint. |
| `organization_not_found` | Supplied `org_id` does not exist. |
| `user_type_not_found` | Supplied `user_type_id` does not exist. |
| `user_not_found` | Requested user does not exist. |
| `user_create_failed` | Insert returned no row. |
| `user_update_failed` | Update returned no row. |
| `user_delete_failed` | Delete returned no row. |
