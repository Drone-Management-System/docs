# Organizations

Organizations represent the institutions and companies participating in the
registry. They map to `caa_registry_api.organization`.

## Permissions

| Operation | Permission |
| --- | --- |
| List or fetch organizations | `organizations.read` |
| Create or replace organizations | `organizations.write` |
| Delete organizations | `organizations.delete` |
| Read organization history | `audit.read` |

## Organization Resource

```json
{
  "org_id": "8c3198d0-6f9d-4a41-b276-6b60da0bdf54",
  "org_name": "Example Airspace Services",
  "organization_kind": "USS",
  "contact_email": "ops@example.test",
  "contact_phone": "+971500000000",
  "country_code": "AE",
  "created_at": "2026-05-14T08:00:00.000Z",
  "updated_at": "2026-05-14T08:00:00.000Z"
}
```

| Field | Type | Notes |
| --- | --- | --- |
| `org_id` | UUID string | Generated by the database. |
| `org_name` | String | Required and unique. |
| `organization_kind` | String | One of `CAA`, `USS`, `OPERATOR`, `SERVICE_PROVIDER`, `OTHER`. |
| `contact_email` | String or `null` | Optional. |
| `contact_phone` | String or `null` | Optional. |
| `country_code` | String or `null` | Uppercased by the API. |
| `created_at` | Timestamp | Database generated. |
| `updated_at` | Timestamp | Updated on replacement. |

## Write Body

```json
{
  "org_name": "Example Airspace Services",
  "organization_kind": "USS",
  "contact_email": "ops@example.test",
  "contact_phone": "+971500000000",
  "country_code": "AE"
}
```

| Field | Required | Validation |
| --- | --- | --- |
| `org_name` | Yes | Non-empty string. |
| `organization_kind` | No | Defaults to `OTHER`; normalized to uppercase; must be supported. |
| `contact_email` | No | Optional string; blank becomes `null`. |
| `contact_phone` | No | Optional string; blank becomes `null`. |
| `country_code` | No | Optional string; blank becomes `null`; non-empty values are uppercased. |

## Endpoint Catalog

| Method | Path | Purpose |
| --- | --- | --- |
| `GET` | `/organizations` | List and search organizations. |
| `GET` | `/organizations/:orgId` | Fetch one organization. |

## `GET /organizations`

<Endpoint
  method="GET"
  path="/organizations"
  title="List organizations"
  permission="organizations.read"
  success="200 OK"
  summary="Returns paginated organizations, optionally filtered by kind and text query."
/>

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "$CAA_REGISTRY_API_BASE_URL/organizations?limit=50&offset=0&q=airspace&organization_kind=USS"
```

### Query Parameters

| Parameter | Default | Maximum | Notes |
| --- | --- | --- | --- |
| `limit` | `50` | `200` | Number of rows to return. |
| `offset` | `0` | `100000` | Number of rows to skip. |
| `q` | None | N/A | Case-insensitive match against `org_name`, `contact_email`, and `country_code`. |
| `organization_kind` | None | N/A | Filter by organization kind. Value is normalized to uppercase. |

### Response

```json
{
  "items": [
    {
      "org_id": "8c3198d0-6f9d-4a41-b276-6b60da0bdf54",
      "org_name": "Example Airspace Services",
      "organization_kind": "USS",
      "contact_email": "ops@example.test",
      "contact_phone": "+971500000000",
      "country_code": "AE",
      "created_at": "2026-05-14T08:00:00.000Z",
      "updated_at": "2026-05-14T08:00:00.000Z"
    }
  ],
  "limit": 50,
  "offset": 0,
  "has_more": false
}
```

Rows are ordered by `created_at desc`, then `org_id desc`.

## `POST /organizations`

<Endpoint
  method="POST"
  path="/organizations"
  title="Create organization"
  permission="organizations.write"
  success="201 Created"
  summary="Creates an organization and writes a create audit event."
/>

```bash
curl -X POST "$CAA_REGISTRY_API_BASE_URL/organizations" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "org_name": "Example Airspace Services",
    "organization_kind": "USS",
    "contact_email": "ops@example.test",
    "contact_phone": "+971500000000",
    "country_code": "ae"
  }'
```

### Request Body

Uses the organization write body documented above.

### Response

Success returns the created row and writes an audit event:

| Audit field | Value |
| --- | --- |
| `entity_type` | `organization` |
| `entity_id` | Created `org_id` |
| `action` | `create` |
| `after_json` | Created organization row |

`country_code` is returned as `AE`.

Uses the organization write body documented above. Replacement updates
`updated_at` to database `now()` and writes before and after

## `DELETE /organizations/:orgId`

<Endpoint
  method="DELETE"
  path="/organizations/:orgId"
  title="Delete organization"
  permission="organizations.delete"
  success="200 OK"
  summary="Deletes an organization when database constraints allow it."
/>

```bash
curl -X DELETE "$CAA_REGISTRY_API_BASE_URL/organizations/8c3198d0-6f9d-4a41-b276-6b60da0bdf54" \
  -H "Authorization: Bearer $TOKEN"
```

### Response

Success returns the deleted row and writes an audit event.

Deletion can be blocked by database constraints when operators still reference
the organization because `operator.org_id` uses `on delete restrict`. Users that
reference the organization are set to `null` by database foreign-key behavior.

## `GET /organizations/:orgId/history`

<Endpoint
  method="GET"
  path="/organizations/:orgId/history"
  title="List organization audit history"
  permission="audit.read"
  success="200 OK"
  summary="Returns audit rows for one organization."
/>

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "$CAA_REGISTRY_API_BASE_URL/organizations/8c3198d0-6f9d-4a41-b276-6b60da0bdf54/history?limit=100"
```

Requires `audit.read`. Returns an array of audit rows for
`entity_type = "organization"` and the supplied `org_id`, ordered newest first.

## Validation Details

| Detail | Trigger |
| --- | --- |
| `org_id_required` | Path `orgId` is missing or blank. |
| `org_name_required` | `org_name` is missing, not a string, or blank. |
| `organization_kind_invalid` | `organization_kind` is not supported. |
| `organization_not_found` | Requested organization does not exist. |
| `organization_create_failed` | Insert returned no row. |
| `organization_update_failed` | Update returned no row. |
| `organization_delete_failed` | Delete returned no row. |
