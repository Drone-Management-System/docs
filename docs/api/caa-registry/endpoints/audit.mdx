# Audit

Audit endpoints expose write history from `public.audit_log`. All audit reads
require `audit.read`.

## Audit Row Shape

```json
{
  "audit_id": "5e47bfa5-ef13-4486-809e-0970b515cb26",
  "entity_type": "uas",
  "entity_id": "faef7df9-972e-4a1d-a6e3-97687b244408",
  "action": "create",
  "actor": "amina@example.test",
  "correlation_id": null,
  "created_at": "2026-05-14T08:00:00.000Z",
  "before_json": null,
  "after_json": {
    "uas_id": "faef7df9-972e-4a1d-a6e3-97687b244408",
    "registration_id": "AE-UAS-0001"
  }
}
```

| Field | Notes |
| --- | --- |
| `audit_id` | UUID primary key generated by the database. |
| `entity_type` | Entity family, such as `organization`, `user`, `operator`, `uas`, `ua`, `operator_type`, or `user_type`. |
| `entity_id` | Entity identifier stored as text. |
| `action` | `create`, `update`, or `delete`. |
| `actor` | Email, username, JWT subject, or `system`. |
| `correlation_id` | Column exists but route handlers do not currently populate it. |
| `created_at` | Database timestamp. |
| `before_json` | Snapshot before update or delete. |
| `after_json` | Snapshot after create, update, or delete. |

## Audit Writes

Mutable endpoints write audit rows automatically:

| API operation | Audit action | Snapshot behavior |
| --- | --- | --- |
| `POST` | `create` | `after_json` contains created row. |
| `PUT` | `update` | `before_json` contains existing row; `after_json` contains updated row. |
| `DELETE` | `delete` | `before_json` and `after_json` contain the deleted row. |

Reference data, organizations, users, operators, UAS records, and UA aircraft
all write audit events for create, update, and delete operations.

## Actor Resolution

The audit helper resolves the actor from the auth context in this order:

1. Email claim.
2. Username claim.
3. JWT subject.
4. `system` if no auth context exists.

For normal API writes, the actor should come from the authenticated token.

## Endpoint Catalog

| Method | Path | Purpose |
| --- | --- | --- |
| `GET` | `/audit` | List and filter audit rows. |
| `GET` | `/organizations/:orgId/history` | Read organization history. |
| `GET` | `/users/:userId/history` | Read user history. |
| `GET` | `/operators/:operatorId/history` | Read operator history. |
| `GET` | `/uas/:uasId/history` | Read UAS history. |
| `GET` | `/ua/:uaId/history` | Read UA aircraft history. |

## `GET /audit`

<Endpoint
  method="GET"
  path="/audit"
  title="List audit rows"
  permission="audit.read"
  success="200 OK"
  summary="Returns paginated audit rows, optionally filtered by entity type, action, and actor."
/>

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "$CAA_REGISTRY_API_BASE_URL/audit?entity_type=uas&action=create&actor=amina@example.test&limit=100&offset=0"
```

### Query Parameters

| Parameter | Default | Maximum | Notes |
| --- | --- | --- | --- |
| `limit` | `100` | `500` | Number of rows to return. |
| `offset` | `0` | `100000` | Number of rows to skip. |
| `entity_type` | None | N/A | Exact match filter. |
| `action` | None | N/A | Exact match filter. |
| `actor` | None | N/A | Exact match filter. |

### Response

```json
{
  "items": [
    {
      "audit_id": "5e47bfa5-ef13-4486-809e-0970b515cb26",
      "entity_type": "uas",
      "entity_id": "faef7df9-972e-4a1d-a6e3-97687b244408",
      "action": "create",
      "actor": "amina@example.test",
      "correlation_id": null,
      "created_at": "2026-05-14T08:00:00.000Z",
      "before_json": null,
      "after_json": {}
    }
  ],
  "limit": 100,
  "offset": 0,
  "has_more": false
}
```

Rows are ordered by `created_at desc`.

## Entity History Routes

History routes return audit rows for one entity. They do not use `offset`; only
`limit` is accepted.

| Route | Entity type used in query | ID parameter |
| --- | --- | --- |
| `/organizations/:orgId/history` | `organization` | `orgId` |
| `/users/:userId/history` | `user` | `userId` |
| `/operators/:operatorId/history` | `operator` | `operatorId` |
| `/uas/:uasId/history` | `uas` | `uasId` |
| `/ua/:uaId/history` | `ua` | `uaId` |

Example:

Rows are ordered by `created_at desc`.

## History Limits

| Query parameter | Default | Minimum | Maximum |
| --- | --- | --- | --- |
| `limit` | `100` | `1` | `500` |

## Validation Details

| Detail | Trigger |
| --- | --- |
| `org_id_required` | Organization history path ID is missing or blank. |
| `user_id_required` | User history path ID is missing or blank. |
| `operator_id_required` | Operator history path ID is missing or blank. |
| `uas_id_required` | UAS history path ID is missing or blank. |
| `ua_id_required` | UA history path ID is missing or blank. |
| `limit_invalid` | `limit` is outside the allowed range or not an integer. |
| `offset_invalid` | `/audit` offset is outside the allowed range or not an integer. |

## Operational Notes

- Audit history is append-only from the perspective of the API. There are no
  endpoints to mutate audit rows.
- The API stores snapshots as JSONB by stringifying the Drizzle-returned row.
- Delete audit events store the deleted row in both `before_json` and
  `after_json`, matching the current implementation.
- `correlation_id` exists in the audit table but is not populated by current
  route handlers.
